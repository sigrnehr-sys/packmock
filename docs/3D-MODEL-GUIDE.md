# 3D 모델 제작 & 임베드 가이드

이 문서는 "내가 직접 만든 봉지 3D 파일을 어떻게 만들고, 어떤 포맷으로,
어디에 넣으면 규격 입력에 맞춰 자동으로 늘어나고 디자인이 입혀지는가"를
설명합니다.

---

## 1. 어떤 파일 포맷? → **glTF Binary (`.glb`)**

웹 3D의 사실상 표준입니다. Three.js / React Three Fiber가 별도 변환 없이
바로 읽고, 텍스처·메시·재질을 한 파일에 담을 수 있어 Vercel 정적 호스팅과
궁합이 좋습니다.

- ✅ `.glb` (권장, 단일 바이너리)
- 🔸 `.gltf + .bin + 텍스처` (가능하지만 파일이 흩어짐 — 비권장)
- ❌ `.fbx`, `.obj`, `.blend`, `.stl` → 그대로는 안 씀. **반드시 `.glb`로 export**

블렌더(Blender) 기준: `File ▸ Export ▸ glTF 2.0 (.glb)`,
Format = **glTF Binary (.glb)**, `+Y Up` 체크.

## 2. 모델링 규칙 (이 규칙을 지켜야 자동 변형이 깨지지 않음)

앱은 GLB의 **기본 형상은 그대로 두고, 사용자가 입력한 규격(mm)에 맞춰
바운딩박스를 비균등 스케일**합니다. "폭이 길어지고 / 높이가 높아지되 형상
균형은 안 깨진다"는 요구가 이렇게 구현됩니다. 그래서 모델은 아래를 지켜야
합니다.

### 2-1. 좌표·크기
- **+Y = 위(높이), X = 폭, Z = 두께(질소 빵빵 방향)**
- 모델 원점(0,0,0)은 봉지의 **기하학적 중심**.
- 기준 치수는 **1 unit = 1 mm 권장**. (예: 240mm 봉지 → 높이 240)
  - 앱은 어차피 입력 규격으로 다시 스케일하므로 절대 크기는 중요치 않지만,
    프로포션(가로:세로:두께 비율)은 그대로 보존됩니다.

### 2-2. 실링영역은 "평평"하게 모델링
- 위/아래(필요시 좌/우) 실링 밴드는 **납작하게(두께 거의 0)** 모델링.
- 가운데 영역만 질소로 **부풀어 오른(bulge)** 형태로.
- 이 평평/부풀음 형태가 GLB에 그대로 들어 있어야 사실적으로 보입니다.
  앱은 비율만 늘릴 뿐, 실링 평탄화를 자동으로 해주지는 않습니다.

### 2-3. UV 펼치기 (디자인이 입혀지는 핵심)
- **앞면(카메라를 향하는 면)** 전체를 하나의 UV 아일랜드로 펼칩니다.
- 그 UV는 **칼선(dieline)과 1:1로 일치**해야 합니다:
  - UV `(0,0)` = 앞면 좌하단, `(1,1)` = 앞면 우상단.
  - 즉 펼친 앞면 전체(실링밴드 포함)가 UV 0~1을 꽉 채움.
  - 디자인 영역은 앱이 `computeLayout()`으로 계산해 텍스처 안에서
    실링 안쪽 사각형에 그려 넣습니다. UV가 칼선과 맞으면 디자인이
    정확히 인쇄 영역에 들어갑니다.
- **뒷면/옆면**은 별도 머티리얼로 분리(아래 2-4 참고). UV는 겹쳐도 무방.

### 2-4. 메시/머티리얼 분리 규칙 (텍스처 자동 매핑용)
GLB 안에서 아래 이름 규칙을 지키면 앱이 자동으로 앞면에만 디자인을 입힙니다.

| 요소 | 메시 이름 | 머티리얼 이름 | 용도 |
| --- | --- | --- | --- |
| 앞면(인쇄면) | `front` | `mat_front` | 업로드 디자인 텍스처가 입혀짐 |
| 뒷면+옆면+실링 | `body` | `mat_foil` | 은박/단색 |

- 메시는 둘로 나누세요(`front`, `body`). 머티리얼 이름을
  `mat_front` / `mat_foil` 로 지정.
- 텍스처를 GLB에 굽지(bake) 마세요. 앱이 런타임에 사용자가 올린
  이미지로 `mat_front`의 baseColor 텍스처를 교체합니다.

## 3. 앱에 임베드하는 법

1. 완성한 GLB를 봉지 종류별 파일명으로 저장:
   - `pillow.glb`, `standup.glb`, `three-side.glb`, `side-gusset.glb`
2. `/public/models/` 폴더에 그대로 넣고 커밋.
3. 끝. (자세한 로더 연결은 아래 "교체 절차" 참고 — 현재는 절차적
   플레이스홀더가 켜져 있음)

### 교체 절차 (플레이스홀더 → 실제 GLB)

현재 `src/components/BagViewer3D.tsx`는 `makePouchGeometry()`로
절차적 메시를 만듭니다. 실제 GLB를 쓰려면 그 `Pouch` 컴포넌트를
아래처럼 GLTF 로더 버전으로 바꾸면 됩니다(드래프트):

```tsx
import { useGLTF } from "@react-three/drei";

function Pouch({ typeId, spec, image }) {
  const { scene } = useGLTF(`/models/${typeId}.glb`);

  // 1) 기본 바운딩박스 → 입력 규격 비율로 비균등 스케일
  // 2) name === "front" 메시의 머티리얼 map 을
  //    buildArtworkCanvas() 로 만든 CanvasTexture 로 교체
  // (computeLayout / buildArtworkCanvas 는 그대로 재사용)

  return <primitive object={scene} />;
}
useGLTF.preload("/models/pillow.glb");
```

`computeLayout()`(칼선)과 `buildArtworkCanvas()`(텍스처)는 2D·3D
공용 모듈이라 GLB로 바꿔도 디자인 매핑 로직은 그대로 재사용됩니다.
플레이스홀더와 실제 GLB가 같은 UV 규칙(2-3)을 따르기 때문입니다.

> 준비되면 알려주세요. GLB 로더 + 비균등 스케일 + 머티리얼 교체까지
> 실제로 연결해 드립니다.

## 4. 모델 제작 체크리스트

- [ ] `.glb` (glTF Binary)로 export, +Y Up
- [ ] 원점 = 봉지 중심, 1 unit ≈ 1 mm
- [ ] 실링밴드 평평 / 가운데 부풀음 형태가 메시에 반영됨
- [ ] 앞면 UV가 칼선과 1:1 (UV 0~1 = 앞면 전체)
- [ ] 메시 분리: `front` / `body`, 머티리얼 `mat_front` / `mat_foil`
- [ ] 텍스처 bake 안 함 (런타임 교체)
- [ ] 폴리곤 < 약 5만 (웹 성능)
- [ ] 파일명 = BagTypeId (`pillow.glb` 등), `/public/models/`에 위치
